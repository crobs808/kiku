## Summary
This release advances Android audio-capture support and release packaging since `v0.1.4-alpha`, including native Android bridge work for system playback capture and updated build/signing automation.

## Diff Summary (since v0.1.4-alpha)

### Added
- Android native activity bridge implementation for permission flow and system playback capture (`MediaProjection` + `AudioPlaybackCaptureConfiguration`).
- Android manifest entries and native activity wiring needed for the new capture bridge.
- Developer command reference for installing the latest signed Android APK.
- Android system-audio validation script: `scripts/dev/validate-android-system-audio.sh`.
- Mobile Android app logo asset pipeline from `assets/kiku-app-logo-mobile.png`.

### Changed
- `kiku-platform` Android capture integration to call into JNI bridge methods for start/stop capture and permission status.
- Desktop/Tauri app logic and UI messaging for Android runtime permission/restart behavior around system audio capture.
- Android release build automation (`scripts/release/build-android.sh`) and documentation (`scripts/release/README.md`) for signed alpha APK output.
- Vendored `third_party/cmake-0.1.58` to stabilize release build dependency behavior.

### Fixed
- Follow-up system-audio stabilization changes across desktop and platform crates after initial Android support landed.
- Build/release flow now consistently produces signed alpha APK artifacts in `target/android/`.

## Known Issues
- Build emits Java source/target 8 deprecation warnings with JDK 21 toolchains.
- Gradle reports deprecated features that are not yet Gradle 9-ready.
- This remains an alpha pre-release build and is not production Play-signing ready.
